# 공부 타임랩스 앱 기획서 (v2)

작성 목적: Claude Code로 직접 개발하기 위한 기준 문서. 이 문서 하나로 콘셉트, 모드, 화면, 디자인, 친구 기능, 기술 구조, 수익 모델, 개발 순서를 파악할 수 있도록 작성함.

개발 전략 요약: 1차로 서버 없는 "혼자하기" 버전을 완성해 스토어에 출시하고, 실제 사용 반응을 확인한 뒤 2차로 서버 기반 "친구와 함께하기"를 붙인다.

---

## 1. 한 줄 소개

공부하는 모습을 촬영하면, 공부 시간에 따라 반응하는 연필 캐릭터가 얹힌 타임랩스 영상을 자동으로 만들어주는 앱.

핵심은 "공부 타이머"가 아니라 "공유하고 싶은 공부 타임랩스를 가장 쉽게 뽑아주는 도구"라는 점이다. 타이머와 타임랩스 촬영은 흔하므로, 촬영을 멈추면 캐릭터가 얹힌 편집 완료 영상이 바로 나온다는 경험으로 차별화한다.

---

## 2. 왜 이 앱인가 (차별점)

- 공부 타이머 앱, 공부 타임랩스 촬영은 이미 많다. 하지만 "촬영 종료 즉시 캐릭터가 얹힌 공유용 영상이 자동 편집되어 나오는" 앱은 없다.
- 차별점은 반응형 캐릭터다. 단순 시간 표시 장식이 아니라, 누적 공부 시간에 따라 표정과 대사가 변한다.
- 캐릭터는 공부의 상징인 연필이다. 이 연필이 사용자를 응원하다가 정작 자기가 먼저 지쳐 잠든다. 이 반전이 공유를 유발하는 핵심 서사다.

바이럴 설계 원리(세 가지를 모두 노림):
- 캡처된다 → 캐릭터가 얹힌 예쁜 결과 화면과 타임랩스 영상
- 자랑거리 → 오래 공부할수록 캐릭터가 "각성"하거나 "먼저 잠듦"(내 연필 재워놓고 나 혼자 공부)
- 공감거리 → 지쳐가는 캐릭터에 사용자가 이입

---

## 3. 타겟 사용자

- 1차: 공부 인증/스터디윗미 문화에 익숙한 중고생, 수험생, 대학생
- 이들은 이미 공부 타임랩스를 찍어 SNS(릴스, 틱톡, 스토리)에 올린다. 이 앱은 그 과정을 원터치로 줄여준다.
- 주의: 타겟에 미성년자가 포함되므로 광고·개인정보·안전 설계에서 아동·청소년 보호 기준을 따라야 한다. (12장, 8장 참고)

---

## 4. 두 가지 모드

앱 실행 후 진입 화면에서 두 모드 중 하나를 고른다.

- 혼자하기: 서버 없이 촬영·시간측정·자동편집·저장까지. 로그인 불필요. (1차 출시 범위)
- 친구와 함께하기: 구글 로그인 후 친구 그룹의 날짜별 피드에 참여. 서버(Supabase) 필요. (2차 확장)

혼자하기만으로도 앱이 완결되게 설계한다. 로그인하기 싫은 사용자도 앱의 핵심 기능을 전부 쓸 수 있어야 한다.

## 5. 핵심 기능

### 5.1 촬영 + 시간 측정
- 앱 안에서 카메라로 공부 모습을 촬영하며 동시에 타이머 작동.
- 촬영 중 화면에 실시간 타이머와, 현재 상태의 캐릭터가 작게 반응하며 가끔 말풍선.
- 정지 버튼으로 촬영·측정 종료.

일시정지:
- 촬영 중 일시정지 가능. (화장실, 잠깐 휴식 등)
- 일시정지 동안은 타이머가 멈추고, 그 시간은 공부 시간에서 제외된다.
- 영상도 정지 전 구간과 재개 후 구간을 하나로 이어 붙인다. (쉬는 구간은 영상에 안 들어감)
- 캐릭터 상태도 "실제 공부한 누적 시간" 기준으로 계산한다. (일시정지 시간 제외)

촬영 방향(세로/가로):
- 촬영 시작 순간의 폰 방향을 감지해 세로면 세로 영상, 가로면 가로 영상으로 찍는다.
- 방향은 촬영 시작 시점에 한 번 정하고 촬영 내내 고정한다. (중간에 폰을 돌려도 영상 방향은 안 바뀜 → 편집 안정성)
- 캐릭터·말풍선·타이머 배치도 그 방향에 맞춰 배치한다. (세로면 캐릭터를 아래/구석, 가로면 옆쪽)
- 결과 영상은 찍은 방향 그대로 나온다. (세로는 릴스·틱톡, 가로는 유튜브·가로 공유에 적합)

### 5.2 반응형 캐릭터
- 누적 공부 시간에 따라 8단계로 변한다. (상세: 6장)
- 말풍선은 두 줄. 윗줄은 실시간 타이머(시:분:초), 아랫줄은 그 시간대 대사.

### 5.3 자동 편집 (타임랩스)
- 촬영이 끝나면 앱이 자동으로 타임랩스 영상을 만든다. (별도 편집 앱 불필요)
- 타임랩스 기본 배속은 고정 제공. 이후 편집 템플릿을 여러 개 추가할 수 있는 구조.
- 영상 안에 시간 경과에 따라 캐릭터가 실제로 변신하는 모습이 담긴다. (쌩쌩→걱정→취침)

### 5.4 공유
- 앱 자체에서 SNS 업로드는 하지 않는다. 완성 영상을 기기 갤러리에 저장하고, 안드로이드 공유 시트로 사용자가 직접 올린다.
- (친구 모드에서는 별도로 친구 피드에 올리는 기능이 추가된다. 8장 참고)

## 7. 화면 구성

모든 화면은 두들링 디자인 가이드(9장)를 따른다.

### 7.1 진입 화면 (모드 선택)
```
+-----------------------------+
|        (종이 질감 배경)       |
|        [ 연필 캐릭터 ]        |
|                             |
|   +---------------------+   |
|   |     혼자하기         |   |
|   +---------------------+   |
|   +---------------------+   |
|   |   친구와 함께하기     |   |  <- 누르면 구글 로그인
|   +---------------------+   |
+-----------------------------+
```

### 7.2 홈 화면
- 큰 캐릭터(기본, pencil_1h), 오늘 총 공부 시간, 연속 기록, 큰 "공부 시작" 버튼
- 하단에 광고 배너 1개 (광고 제거 구매 시 사라짐)

### 7.3 촬영 화면 (세로/가로 자동)
- 카메라 프리뷰, 실시간 타이머, 현재 상태 캐릭터+말풍선, 일시정지/정지 버튼
- 광고 없음 (공부 방해 금지)

### 7.4 결과 화면
- 타임랩스 영상 미리보기(캐릭터 변신 포함)
- 요약 카드: 총 공부 시간 + 캐릭터 최종 상태와 한마디
- "갤러리 저장", "공유" 버튼 / 친구 모드면 "친구 피드에 올리기" 버튼 추가
- 이 화면을 보고 넘어갈 때 가끔 전면 광고 (광고 제거 구매 시 없음)
- 캡처 포인트라 가장 공들여 디자인

### 7.5 친구 피드 화면 (친구 모드)
- 날짜별로 묶인 피드(스터디 다이어리 느낌). 위→아래로 오늘, 어제, 그저께…
- 각 카드: 닉네임, 그날 공부 시간, 캐릭터 최종 상태, 타임랩스 영상, 스탬프 반응
```
+-----------------------------+
|  ─────  오늘  ─────          |
|  [ 민수  06:12  "독하다.." ]  |
|  [ 영상 ]        👍 🔥        |
|                             |
|  [ 지은  03:40  "화이팅!" ]   |
|  [ 영상 ]        👍          |
|  ─────  어제  ─────          |
|  ...                        |
+-----------------------------+
```

## 8. 친구 기능 상세 (2차 확장)

### 8.1 로그인
- 구글 로그인 사용. Supabase Authentication으로 구현.
- 친구 모드 진입 시에만 로그인. 혼자하기는 로그인 불필요.

### 8.2 비공개 그룹 + 초대 링크
- 친구 그룹은 완전 비공개. 검색으로 아무도 못 찾는다.
- 초대 링크를 받은 사람만 해당 그룹에 들어올 수 있다. (링크 = 유일한 입장 경로)
- 미성년자 안전을 위한 핵심 설계. 모르는 사람에게 영상이 노출되지 않는다.

### 8.3 날짜별 피드
- 친구가 올린 공부 타임랩스를 날짜별로 모아 보여준다.
- 올리기는 선택이다. 결과 화면에서 "친구 피드에 올리기"를 누른 것만 올라간다. (강제 공유 금지)

### 8.4 스탬프 반응
- 좋아요/"대단해"/"불타오른다" 같은 간단한 스탬프만. (댓글 없음 → 구현 단순 + 안전)

### 8.5 데이터 흐름
- 올릴 때: 경량화(저화질) 영상을 저장소에 업로드 → DB에 (작성자, 날짜, 공부시간, 캐릭터 최종상태, 영상 주소, 그룹ID) 기록.
- 볼 때: 내 그룹의 날짜별 기록을 DB에서 불러와, 영상 주소로 영상을 가져와 보여줌.
- 원본 고화질 타임랩스는 기기 갤러리에만 저장. 피드용은 가벼운 버전만 업로드. (비용 절감)

## 9. UI 디자인 가이드 (중요)

전체 콘셉트: 흰 종이 위에 검은 연필로 직접 그린 손그림(두들링). 캐릭터가 연필 두들링이므로 앱 전체가 같은 노트/스케치북 세계관 안에 있는 느낌을 준다.

### 9.1 색
| 용도 | 색상 | 비고 |
|---|---|---|
| 배경 | #FAF7EF | 아주 살짝 누런 흰색, 종이 질감 |
| 잉크(선/기본 글자) | #2B2B2B | 완전한 검정 대신 부드러운 먹색 |
| 보조 글자 | #6B6B6B | 연필 회색, 캡션용 |
| 포인트(절제) | #F4CB7E | 연필 나무색. 강조 최소화 |
| 볼터치 분홍 | #F4A9A9 | 캐릭터/아주 작은 강조에만 |

화면 대부분은 종이색 배경 + 먹색 선/글자로만 구성한다.

### 9.2 선과 형태 (두들링 핵심)
- 모든 테두리·구분선·아이콘은 손으로 그린 듯 살짝 삐뚤빼뚤하게. 자로 잰 직선/원 금지.
- 선 굵기 2~3px, 끝은 둥글게. 굵기가 미세하게 변하면 더 좋다.
- 버튼은 손그림 둥근 사각형 테두리, 안은 비우거나 아주 옅은 종이색.
- 구현 팁: 이중선 겹치기, SVG 경로 미세 흔들림, 손그림 테두리 나인패치 등으로 손맛을 낸다.

### 9.3 폰트
- 한글은 손글씨 느낌 무료 폰트(나눔손글씨, 배민 한나/주아 계열 등, 라이선스 확인).
- 숫자(타이머)는 너무 흘리지 않고 둥근 손그림 느낌.
- 제목/버튼은 손글씨, 긴 설명은 조금 더 읽기 쉬운 폰트로 역할 분리 가능.

### 9.4 모션
- 절제한다. 과한 전환은 AI/템플릿 느낌을 준다.
- 허용: 상태 전환 시 캐릭터가 살짝 통통, 버튼 누를 때 약간 흔들림.
- 촬영 화면·결과 카드 캐릭터는 Lottie로 부드럽게. (11장)

### 9.5 금지 사항 (반드시 지킬 것)
- 그라데이션 금지.
- 매끄러운 머티리얼 곡선/드롭섀도 남발 금지. 그림자가 필요하면 손 빗금(해칭)으로 대체.
- 형광색, 채도 높은 원색, 네온 금지.
- 사진 배경, 스톡 이미지, 3D 렌더 느낌 금지.
- 기본 아이콘 라이브러리 그대로 쓰기 금지. 아이콘도 손그림으로.

## 10. 기술 구조

### 10.1 플랫폼 전략
- 1차: 안드로이드 네이티브(Kotlin), 구글 플레이 출시.
- iOS는 이후 확장. 아래처럼 계층을 분리해 영상 엔진만 다시 구현하면 되도록 설계.

### 10.2 계층 분리 (iOS 확장 대비)
- UI/화면/타이머/캐릭터 상태·대사 로직 → 이식 가능한 순수 로직.
- 카메라 촬영 + 영상 합성 엔진 → 플랫폼 전용(지금 Kotlin, iOS는 Swift로 교체).
- 두 층 사이를 인터페이스로 끊어둔다.

### 10.3 안드로이드 기술
- 카메라: CameraX (방향 감지·고정, 일시정지/재개 포함)
- 영상 합성/인코딩: Media3 (Jetpack). 구글 유지보수라 라이브러리 종료 위험 적음. 오버레이 합성·재인코딩 정식 지원.
- 실시간 캐릭터 애니메이션: Lottie (촬영 화면·결과 카드)
- 로컬 저장: Room (공부 기록, 연속 일수, 광고 제거 구매 상태 등)
- 저장/공유: MediaStore 갤러리 저장 + 안드로이드 공유 시트

### 10.4 모드별 백엔드
- 혼자하기: 서버리스. 모든 처리가 기기 안에서 끝난다. 서버 비용 0원.
- 친구와 함께하기: Supabase 사용.
  - Supabase Authentication: 구글 로그인
  - Supabase Database(Postgres): 사용자·친구그룹·피드 기록. 그룹은 링크 기반 비공개, Row Level Security로 남의 그룹 접근 차단.
  - 영상 저장: Supabase Storage 또는 Cloudflare R2. 비용상 R2 권장(전송 요금 0원). Supabase는 로그인·DB만 맡기는 구조가 유리.

## 11. 타임랩스 합성 방식 (핵심 구현)

타임랩스는 프레임을 띄엄띄엄 뽑아 빠르게 잇는다. 최종 영상을 프레임 단위로 조립할 때, 각 프레임에 아래를 겹쳐 그린다.

1. 그 시점 촬영 프레임
2. 그 시점 누적 시간에 해당하는 캐릭터 상태 이미지(pencil_Nh.png)
3. 말풍선 + 그 시점 경과 시간과 대사

이렇게 하면 완성 영상을 빨리 감을 때 캐릭터가 쌩쌩→걱정→취침으로 변신하는 것처럼 보인다. 이 "변신 타임랩스"가 사용자가 남에게 보여주고 싶은 결정적 장면이다.

주의:
- 영상에 굽는 캐릭터는 상태별 정지 이미지(또는 2~3장 번갈아). 부드러운 Lottie를 영상에 그대로 굽는 것은 무겁다.
- 진짜 Lottie 애니메이션은 촬영 화면·결과 카드(실시간 UI)에서만. 체감 차이는 거의 없고 난이도는 크게 낮아진다.
- 일시정지 처리: 조립 시 정지 구간 프레임은 건너뛰고, 시간 계산도 정지 시간을 뺀 누적으로 한다.

## 12. 수익 모델 및 비용

### 12.1 광고 (애드몹)
- 앱 광고는 애드센스(AdSense)가 아니라 애드몹(AdMob)을 쓴다. (애드센스는 웹사이트용. 앱 소개 웹페이지를 따로 만들 때만 애드센스 사용.)
- 위치: 홈 화면 하단 배너 1개 + 결과 화면을 보고 넘어갈 때 가끔 전면 광고. 촬영 화면에는 광고를 절대 넣지 않는다.
- 미성년자 경고(중요): 타겟에 중고생이 포함되어, 앱 등록 시 아동·청소년 대상 여부에서 걸린다. 대상에 해당하면 맞춤형 광고가 제한되어 단가(eCPM)가 낮아지고 광고 형식도 좁아진다. → 광고 수익이 기대보다 적을 수 있으니 광고에만 의존하지 않는다.

### 12.2 광고 제거 (인앱 결제)
- 반드시 구글 인앱 결제(Google Play Billing)를 쓴다. 외부 결제(토스·카카오페이 직접 연동)를 앱 안에 붙이면 정책 위반으로 앱이 내려갈 수 있다.
- 상품: 광고 제거, 가격 1,900원, 일회성 구매(비소모성).
- 구매하면 앱 전체에서 광고 제거.
- 수수료: 매출 일정 구간까지 구글이 약 15%. 1,900원 판매 시 실수령 약 1,600원.
- 구매 상태: 기기 로컬 저장 + 친구 로그인 사용자는 계정에도 연동(기기 변경 시 유지).

### 12.3 서버 비용 (친구 모드에만 발생)
- 혼자하기: 서버 없음 → 0원.
- Supabase 무료: 파일 저장 1GB, 전송(egress) 월 5GB 수준. 소수 친구 테스트용. 7일 미사용 시 프로젝트 자동 정지.
- Supabase Pro(약 월 $25): 저장 100GB, 전송 월 250GB 수준. 초과 시 전송 GB당 약 $0.09.
- 진짜 병목은 저장이 아니라 전송량(친구들이 영상을 볼 때마다 소비). 그래서 영상은 전송 요금이 없는 Cloudflare R2에 두는 구조를 권장.
- 절감책: 피드용 영상은 저화질 경량 버전만 업로드. 오래된 피드 영상은 자동 삭제(예: 최근 N일만 보관)해 저장·전송을 일정하게 유지.
- (요금은 2026년 기준 추정치. 실제 값은 supabase.com/pricing에서 확인할 것.)

### 12.4 손익 개요 (추정)
- 무료 구간(0~소수 사용자): 비용 0원.
- 일 활성 약 100명: 광고 월 $30~90 추정 vs Supabase Pro $25 → 대체로 커버.
- 규모가 커질수록 광고 + 인앱결제가 서버비를 상회.
- 단, 미성년자 광고 제약으로 광고 추정치는 하향 가능. 광고 제거 인앱결제와 향후 캐릭터 스킨 등 광고 외 수익을 병행 권장.

## 14. 개발 순서 (마일스톤)

각 단계마다 "돌아가는 결과물"이 나오도록 한다.

1차 출시 (혼자하기, 서버 없음)
1. 뼈대: 타이머 + 카메라 촬영 + 영상 저장. (+ 일시정지, + 방향 자동 감지·고정)
2. 타임랩스 변환: 촬영 영상을 배속 타임랩스로 자동 변환. (일시정지 구간 제외)
3. 캐릭터 로직: 누적 시간 → 이미지/대사 선택, 촬영 화면에 실시간 캐릭터+말풍선.
4. 영상 합성: 타임랩스 각 프레임에 캐릭터+말풍선 오버레이(변신 타임랩스 완성).
5. 화면 구현: 진입/홈/촬영/결과 화면을 두들링 가이드대로.
6. 결과 화면 다듬기 + 갤러리 저장/공유.
7. 애니메이션: Lottie로 실시간 캐릭터 생동감.
8. 수익: 애드몹(배너+전면) + 인앱결제(광고 제거 1,900원) 연동.
9. 출시: 권한 안내(카메라·저장), 아이콘/스플래시, 아동·청소년 대상 여부 설정, 스토어 등록.

2차 확장 (친구와 함께하기, 서버)
10. 백엔드: Supabase 프로젝트, 구글 로그인, DB 스키마(사용자·그룹·피드), RLS로 그룹 격리, 영상 저장소(R2 권장) 연결.
11. 친구 기능 화면: 초대 링크 그룹 참여, 날짜별 피드, 스탬프 반응, "친구 피드에 올리기"(경량 영상 업로드).
12. 친구 기능 출시(2차) 및 안전 정책(신고·차단, 개인정보 처리방침) 정비.

## 15. 비고

- 캐릭터는 사용자(우진) 본인이 그린 원본 두들링이므로 저작권은 본인에게 있다. 스토어 출시에 문제없다.
- 다른 회사/브랜드 마스코트를 닮게 그리지 않는다. (상표권 문제로 앱이 내려갈 수 있음)
- 권한 안내(카메라·저장) 문구는 두들링 톤으로 작성.
- 미성년자 대상 앱은 개인정보 처리방침, 아동·청소년 보호, 부적절 콘텐츠 신고·차단이 특히 중요하다. 친구 기능 출시 전 반드시 정비.
- 확장 여지: 편집 템플릿 추가, 캐릭터 스킨/표정 추가, 통계 리포트, 화이트노이즈 등은 출시 후 반응을 보고 추가.
