# 공부 타임랩스 앱 기획서

작성 목적: Claude Code로 직접 개발하기 위한 개발 기준 문서. 이 문서 하나로 앱의 콘셉트, 화면, 디자인, 기술 구조, 개발 순서를 파악할 수 있도록 작성함.

---

## 1. 한 줄 소개

공부하는 모습을 촬영하면, 공부 시간에 따라 반응하는 연필 캐릭터가 얹힌 타임랩스 영상을 자동으로 만들어주는 앱.

핵심은 "공부 타이머"가 아니라 "공유하고 싶은 공부 타임랩스를 가장 쉽게 뽑아주는 도구"라는 점이다. 타이머와 타임랩스 촬영은 이미 흔하므로, 이 앱은 촬영을 멈추면 캐릭터가 얹힌 편집 완료 영상이 바로 나온다는 경험으로 차별화한다.

---

## 2. 왜 이 앱인가 (차별점)

- 공부 타이머 앱, 공부 타임랩스 촬영은 이미 많다. 하지만 "촬영 종료 즉시 캐릭터가 얹힌 공유용 영상이 자동 편집되어 나오는" 앱은 없다.
- 차별점은 캐릭터다. 단순히 시간을 표시하는 장식이 아니라, 누적 공부 시간에 따라 표정과 대사가 변하는 반응형 캐릭터다.
- 캐릭터는 공부의 상징인 연필이다. 그런데 이 연필이 사용자를 응원하다가 정작 자기가 먼저 지쳐 잠든다. 이 반전이 공유를 유발하는 핵심 서사다.

바이럴 설계 원리 (세 가지를 모두 노린다):
- 결과물이 캡처된다 → 캐릭터가 얹힌 예쁜 결과 화면과 타임랩스 영상
- 자랑거리가 된다 → 오래 공부할수록 캐릭터가 "각성"하거나 "먼저 잠듦" (내 연필 재워놓고 나 혼자 공부)
- 공감거리가 된다 → 지쳐가는 캐릭터에 사용자가 이입

---

## 3. 타겟 사용자

- 1차: 공부 인증/스터디윗미 문화에 익숙한 중고생, 수험생, 대학생
- 이들은 이미 공부 타임랩스를 찍어 SNS(인스타 릴스, 틱톡, 스토리)에 올린다. 이 앱은 그 과정을 원터치로 줄여준다.

---

## 4. 핵심 기능

### 4.1 촬영 + 시간 측정
- 앱 안에서 카메라로 공부하는 모습을 촬영하며 동시에 타이머가 작동한다.
- 촬영 중 화면에는 실시간 타이머와, 현재 상태의 캐릭터가 작게 반응하며 가끔 말풍선을 띄운다.
- 정지 버튼을 누르면 촬영과 시간 측정이 종료된다.

### 4.2 반응형 캐릭터
- 캐릭터는 누적 공부 시간에 따라 8단계로 변한다. (상세: 5장)
- 말풍선은 두 줄 구조다. 윗줄은 실시간 타이머(시:분:초), 아랫줄은 그 시간대의 대사.

### 4.3 자동 편집 (타임랩스)
- 촬영이 끝나면 앱이 자동으로 타임랩스 영상을 만든다. (별도 편집 앱 불필요)
- 타임랩스 기본 배속은 고정 제공. 이후 편집 템플릿을 여러 개 추가할 수 있는 구조로 설계.
- 영상 안에는 시간 경과에 따라 캐릭터가 실제로 변신하는 모습이 담긴다. (쌩쌩 → 걱정 → 취침)

### 4.4 공유
- 앱 자체에서 SNS 업로드는 하지 않는다. 완성된 영상을 기기 갤러리에 저장하고, 안드로이드 공유 시트를 통해 사용자가 직접 원하는 곳(인스타, 틱톡 등)에 올린다.
- 서버가 필요 없는 이유이기도 하다. (7장 참고)

## 5. 캐릭터 시스템

### 5.1 캐릭터 에셋
- 8개 상태의 캐릭터 이미지가 준비되어 있다. (연필 두들링 스타일, 배경 투명 PNG)
- 파일명: pencil_1h.png ~ pencil_8h.png
- 규격: 모든 이미지가 동일 캔버스(가로 359 x 세로 404 px)에 발 위치 기준으로 정렬되어 있다. 따라서 상태가 바뀌어도 화면상 캐릭터가 튀지 않고 제자리에 고정된다.
- 말풍선은 이미지에 포함되어 있지 않다. 앱이 캐릭터 위쪽에 말풍선과 글자를 직접 그려 넣는다. (글자 수에 따라 말풍선 크기가 자동으로 늘어나도록 구현)

### 5.2 8단계 상태와 표정

| 단계 | 이미지 | 표정/자세 | 감정 |
|---|---|---|---|
| 1시간째 | pencil_1h | 손 흔들며 인사 | 쌩쌩한 응원 |
| 2시간째 | pencil_2h | 주먹 쥐고 응원 (반짝 효과) | 흐뭇한 응원 |
| 3시간째 | pencil_3h | 차분한 미소 (반짝이) | 감탄 |
| 4시간째 | pencil_4h | 두 팔 번쩍 (절정) | 최고 텐션 |
| 5시간째 | pencil_5h | 땀 흘리며 시무룩 | 의아함 (전환점) |
| 6시간째 | pencil_6h | 더 지쳐 축 처짐 | 걱정 |
| 7시간째 | pencil_7h | 눈 감기 직전, 꾸벅꾸벅 | 졸림 |
| 8시간째 | pencil_8h | 베개 베고 취침 | 취침 |

감정 곡선: 1~4시간은 응원이 점점 커지는 상승, 4시간이 정점, 5시간부터 걱정으로 전환, 7~8시간에 캐릭터가 방전. 사용자는 계속 공부하는데 캐릭터가 먼저 지치는 대비가 핵심이다.

### 5.3 시간대별 말풍선 대사

정시마다 대사가 바뀌고, 30분 단위로 중간 대사가 추가되어 캐릭터가 계속 말을 거는 느낌을 준다. 캐릭터 이미지는 8장이므로, 같은 정시 구간(예: 1시간~1시간59분)은 동일 이미지를 쓰고 대사 텍스트만 교체한다.

| 누적 시간 | 대사 | 사용 이미지 |
|---|---|---|
| 00:00~ | 자, 시작해볼까! | pencil_1h |
| 00:30~ | 오, 집중 들어간다 | pencil_1h |
| 01:00~ | 제대로 공부 시작했군.. | pencil_1h |
| 01:30~ | 이 페이스 좋은데? | pencil_1h |
| 02:00~ | 잘하고 있어! 계속해 | pencil_2h |
| 02:30~ | 슬슬 몸이 풀렸지? | pencil_2h |
| 03:00~ | 집중력 짱이다! 대단해 | pencil_3h |
| 03:30~ | 여기서 조금만 더! | pencil_3h |
| 04:00~ | 화이팅! 화이팅! | pencil_4h |
| 04:30~ | 어라, 아직도 하네? | pencil_4h |
| 05:00~ | 계속 하는거야..? | pencil_5h |
| 05:30~ | 나 슬슬 힘든데.. | pencil_5h |
| 06:00~ | 무리하는거 아니지? | pencil_6h |
| 06:30~ | 너 진짜 독하다.. | pencil_6h |
| 07:00~ | 나 자러가도 돼...? | pencil_7h |
| 07:30~ | 눈이.. 감긴다.. | pencil_7h |
| 08:00~ | Zzz... | pencil_8h |
| 09:00~ | (쿨쿨.. 넌 계속 가는구나) | pencil_8h |

구현 로직:
- 사용 이미지 = 누적 시간(시) 값을 내림한 뒤 1~8로 제한. (8시간 이상은 계속 pencil_8h)
- 대사 = 30분 단위 구간으로 위 표에서 선택.
- 8시간째 대사의 이모지(🌛 등)는 기기마다 모양이 달라 쓰지 않는다. "Zzz..."처럼 직접 그리는 텍스트로 처리. (8h 이미지에 이미 Z 표시가 그려져 있음)

## 6. 화면 구성

화면은 세 개로 충분하다. 모든 화면은 8장의 디자인 가이드(두들링 스타일)를 따른다.

### 6.1 홈 화면
- 큰 캐릭터 (기본 상태, 인사하는 pencil_1h)
- 오늘 총 공부 시간, 연속 기록(며칠 연속)
- 큰 "공부 시작" 버튼

```
+-----------------------------+
|            (종이 질감 배경)   |
|                             |
|        [ 연필 캐릭터 ]        |
|         손 흔드는 그림         |
|                             |
|   오늘 공부   03:24:10        |
|   연속       5일             |
|                             |
|   +---------------------+   |
|   |     공부 시작        |   |   <- 손그림 테두리 버튼
|   +---------------------+   |
+-----------------------------+
```

### 6.2 촬영 화면
- 카메라 프리뷰(공부하는 모습)
- 상단 또는 모서리에 실시간 타이머
- 화면 한쪽에 현재 상태의 캐릭터가 작게 반응, 가끔 말풍선
- 정지 버튼

```
+-----------------------------+
|  ● REC       01:12:33        |   <- 실시간 타이머
|                             |
|      (카메라 프리뷰)          |
|                             |
|                   ( 말풍선 ) |
|                    캐릭터     |   <- 현재 상태로 반응
|                             |
|          (  정지  )          |
+-----------------------------+
```

### 6.3 결과 화면
- 방금 만든 타임랩스 영상 미리보기 (캐릭터 변신이 담긴)
- 요약 카드: 총 공부 시간 + 캐릭터의 최종 상태와 한마디
- "갤러리 저장", "공유" 버튼
- 이 화면이 사용자가 캡처하는 순간이므로 가장 공들여 디자인한다.

```
+-----------------------------+
|      오늘의 공부 기록          |
|                             |
|     [ 타임랩스 영상 재생 ]     |
|                             |
|   총 공부      08:05:22       |
|   [ 캐릭터: 먼저 잠들어버림 ]   |
|   "Zzz... 넌 계속 가는구나"    |
|                             |
|  [ 갤러리 저장 ]  [ 공유 ]     |
+-----------------------------+
```

## 7. UI 디자인 가이드 (중요)

전체 콘셉트: 흰 종이 위에 검은 연필로 직접 그린 손그림(두들링). 캐릭터가 연필 두들링이므로, 앱 전체가 같은 노트/스케치북 세계관 안에 있는 느낌을 준다.

### 7.1 색

| 용도 | 색상 | 비고 |
|---|---|---|
| 배경 | #FAF7EF | 아주 살짝 누런 흰색, 종이 질감 |
| 잉크 (선/기본 글자) | #2B2B2B | 완전한 검정(#000) 대신 살짝 부드러운 먹색 |
| 보조 글자 | #6B6B6B | 연필 회색, 설명/캡션용 |
| 포인트 (아주 절제) | #F4CB7E | 연필 나무색. 강조는 최소한만 |
| 볼터치 분홍 | #F4A9A9 | 캐릭터/아주 작은 강조에만 |

- 색은 위 팔레트 밖으로 나가지 않는다. 화면 대부분은 종이색 배경 + 먹색 선/글자로만 구성한다.

### 7.2 선과 형태 (두들링 핵심)
- 모든 테두리, 구분선, 아이콘은 손으로 그린 듯 살짝 삐뚤빼뚤한 선으로 그린다. 자로 잰 듯한 완벽한 직선/원을 쓰지 않는다.
- 선 굵기 2~3px, 끝은 둥글게(round cap). 완벽하게 균일하지 않고 미세하게 굵기가 변하면 더 좋다.
- 버튼은 손그림 둥근 사각형 테두리로 만들고, 안은 채우지 않거나 아주 옅은 종이색으로만 채운다. (종이에 펜으로 박스를 그린 느낌)
- 실제 구현 팁: 손그림 느낌은 테두리를 살짝 이중선으로 겹치거나, SVG 경로에 미세한 흔들림을 주거나, 손그림 테두리 이미지를 나인패치로 쓰는 방식으로 낸다.

### 7.3 폰트
- 한글은 손글씨 느낌의 무료 폰트를 쓴다. (예: 나눔손글씨 계열, 배민 한나/주아체 계열 등 라이선스 확인 후 사용)
- 숫자(타이머)는 가독성을 위해 너무 흘림체가 아닌, 둥글고 손그림 느낌이 나는 폰트를 쓴다.
- 제목/버튼은 손글씨 폰트, 긴 설명은 조금 더 읽기 쉬운 폰트로 역할을 나눠도 된다.

### 7.4 모션
- 애니메이션은 절제한다. 과한 전환 효과는 오히려 AI/템플릿 느낌을 준다.
- 허용: 상태가 바뀔 때 캐릭터가 살짝 통통 튀는 정도, 버튼 누를 때 아주 약간 흔들리는 정도.
- 촬영 화면과 결과 카드의 캐릭터는 Lottie로 부드럽게 움직이게 한다. (10장 참고)

### 7.5 금지 사항 (반드시 지킬 것)
- 그라데이션 금지.
- 매끄러운 머티리얼 디자인 곡선/그림자(drop shadow) 남발 금지. 그림자를 쓸 거면 손으로 빗금 친 듯한 해칭으로 대체.
- 형광색, 채도 높은 원색, 네온 금지.
- 사진 배경, 스톡 이미지, 3D 렌더 느낌 금지.
- 완벽하게 매끈한 벡터 아이콘 세트(라이브러리 기본 아이콘) 금지. 아이콘도 손그림으로.

## 8. 기술 구조

### 8.1 플랫폼 전략
- 1차 목표: 안드로이드 네이티브 (Kotlin). 스토어(구글 플레이) 출시.
- iOS는 이후 확장. 지금은 아래처럼 계층을 분리해서, 나중에 영상 엔진만 다시 구현하면 되도록 설계한다.

### 8.2 계층 분리 (iOS 확장 대비)
- UI / 화면 / 타이머 / 캐릭터 상태·대사 선택 로직 → 이식 가능한 순수 로직. (플랫폼 독립)
- 카메라 촬영 + 영상 합성 엔진 → 플랫폼 전용. (지금은 Kotlin, iOS 확장 시 Swift로 교체)
- 이 두 층 사이를 인터페이스로 끊어둔다.

### 8.3 사용 기술 (안드로이드)
- 카메라: CameraX
- 영상 합성/인코딩: Media3 (Jetpack). 구글이 직접 유지보수하므로 라이브러리 종료 위험이 적다. 영상 위 오버레이(캐릭터, 말풍선) 합성과 재인코딩을 정식 지원.
- 실시간 캐릭터 애니메이션: Lottie (촬영 화면, 결과 카드)
- 로컬 데이터 저장: Room (공부 기록, 연속 일수 등)
- 저장/공유: MediaStore로 갤러리 저장 + 안드로이드 공유 시트

### 8.4 서버리스
- 로그인, 클라우드, 백엔드 서버가 필요 없다. 촬영·측정·편집·저장이 모두 기기 안에서 처리되고, 공유는 사용자가 직접 한다.
- 따라서 서버 유지비 0원. 데이터는 기기 내부 저장소에만 둔다.

## 9. 타임랩스 합성 방식 (핵심 구현)

타임랩스는 프레임을 띄엄띄엄 뽑아 빠르게 잇는 방식이다. 최종 영상을 만들 때 프레임을 하나씩 조립하는데, 각 프레임을 그릴 때 아래를 겹쳐 그린다.

1. 그 시점 촬영 프레임
2. 그 시점 누적 시간에 해당하는 캐릭터 상태 이미지 (pencil_Nh.png)
3. 말풍선 + 그 시점의 경과 시간과 대사

이렇게 하면 완성된 영상을 빨리 감을 때 캐릭터가 저절로 쌩쌩 → 걱정 → 취침으로 변신하는 것처럼 보인다. 이 "변신 타임랩스"가 사용자가 남에게 보여주고 싶은 결정적 장면이다.

주의:
- 영상 안(구운 영상)에 넣는 캐릭터는 상태별 정지 이미지(또는 2~3장 번갈아 그린 간단한 움직임)로 처리한다. 부드러운 Lottie 애니메이션을 영상에 그대로 굽는 것은 무겁다.
- 진짜 부드러운 Lottie 애니메이션은 촬영 화면과 결과 카드(실시간 UI)에서만 보여준다. 사용자 체감 차이는 거의 없고 개발 난이도는 크게 낮아진다.

## 10. 에셋 목록

- 캐릭터: pencil_1h.png ~ pencil_8h.png (투명 배경, 359 x 404 px, 발 위치 정렬)
- 말풍선: 이미지가 아닌 앱에서 손그림 스타일로 직접 그림. 글자 수에 따라 크기 자동 조절.
- 폰트: 손글씨 계열 한글 폰트 + 숫자용 둥근 폰트 (라이선스 확인 후 번들)
- 아이콘: 손그림 스타일로 직접 제작 (기본 아이콘 라이브러리 사용 금지)
- (선택) Lottie 파일: 실시간 화면용 캐릭터 애니메이션. 초기에는 정지 이미지로 시작하고 이후 추가 가능.

## 11. 개발 순서 (마일스톤)

각 단계마다 "돌아가는 결과물"이 나오도록 순서를 잡는다.

1. 뼈대: 타이머 + 카메라 촬영 + 영상 저장. "찍고 멈추면 영상이 갤러리에 저장된다"까지.
2. 타임랩스 변환: 촬영 영상을 배속 타임랩스로 자동 변환.
3. 캐릭터 로직: 누적 시간 → 이미지/대사 선택 로직 구현, 촬영 화면에 실시간 캐릭터+말풍선 표시.
4. 영상 합성: 타임랩스 각 프레임에 캐릭터+말풍선 오버레이 (변신 타임랩스 완성).
5. 세 화면 완성: 홈 / 촬영 / 결과 화면을 두들링 디자인 가이드대로 구현.
6. 결과 화면 다듬기: 요약 카드, 저장/공유 기능. (캡처 포인트라 가장 공들임)
7. 애니메이션: Lottie로 실시간 캐릭터 생동감 추가.
8. 출시 준비: 권한 안내(카메라·저장), 아이콘/스플래시, 스토어 등록.

## 12. 비고

- 캐릭터는 사용자(우진) 본인이 그린 원본 두들링이므로 저작권은 본인에게 있다. 스토어 출시에 문제없다.
- 다른 회사/브랜드의 마스코트를 닮게 그리지 않는다. (상표권 문제로 앱이 내려갈 수 있음)
- 권한: 카메라, 저장소(미디어) 접근 권한 안내 문구를 두들링 톤으로 작성.
- 확장 여지: 편집 템플릿 추가, 캐릭터 스킨/표정 추가, 통계 리포트, 배경음/화이트노이즈 등은 출시 후 반응을 보고 추가.
